# Topicus Web App (University Project)

A web application developed as part of a university project, inspired by real-world systems used in industry. The project focuses on applying software engineering principles such as modular architecture, separation of concerns, and client-server communication.

---

## Overview

This project was built in a university setting to explore how modern web applications are structured and implemented. It demonstrates the integration of frontend and backend components, along with handling data flow between them.

The repository represents a work-in-progress system and is not intended to be a fully production-ready application.

---

## Features

- Structured web application with frontend and backend separation
- Backend handling of application logic and request processing
- Frontend for user interaction and data presentation
- Communication between frontend and backend via APIs
- Basic data flow and state handling

Note: Some features are incomplete due to project scope and time constraints.

---

## Architecture

The project follows a layered architecture:

- Frontend  
  Responsible for user interaction and presentation. Communicates with the backend.

- Backend  
  Handles request processing, application logic, and coordination of data flow.

- Data Layer  
  Manages storage and retrieval of application data.

This separation improves maintainability, scalability, and clarity of the system.

---

## Tech Stack

- Java (backend)
- Web technologies (HTML, CSS, JavaScript)
- REST-style communication
- Git for version control
